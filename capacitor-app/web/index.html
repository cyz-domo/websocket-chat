Capacitor shell is ready.

The app defaults to https://chat.6143443.xyz/chat/login/. You can still override it with CAP_SERVER_URL before running npx cap sync.

Examples

npm run sync:prod

CAP_SERVER_URL=http://10.0.2.2:8000/chat/login/ npm run sync

CAP_SERVER_URL=https://your-domain/chat/login/ npm run sync

Why this setup

The Django pages stay as the source of truth, while Capacitor gives us native packaging, push notifications, camera access, and store-ready iOS/Android projects.
